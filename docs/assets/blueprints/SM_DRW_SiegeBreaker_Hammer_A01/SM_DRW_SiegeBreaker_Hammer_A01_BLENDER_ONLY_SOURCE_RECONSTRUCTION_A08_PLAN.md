## Flamestrike Direction

Flamestrike prohibited TRELLIS and every other image-generation or image-to-3D
system from this process. Blender is the sole approved visual and geometry
construction exception. Flamestrike then directed Codex to apply the process
proven successful on the Blood Axe Cairnstone A005 A04 plinth to Siege Breaker.

## Proven Method Authority

The Cairnstone A005 A04 plinth proves this bounded method:

1. lock the unchanged source and exact numeric constraints;
2. construct one fresh component in Blender from the approved source and
   measurements;
3. consume no rejected or quarantined predecessor geometry;
4. present the component against its source-owned visual region;
5. let Flamestrike approve, revise, or reject visual fidelity;
6. preserve an approved component exactly while the next component is built.

The A04 plinth is an `authoritative visual reference`. The complete A04
Cairnstone candidate was rejected because other base components were wrong;
that rejection does not invalidate the approved plinth method.

## Component Gate Rule

For every component:

- create a fresh isolated Blender collection;
- preserve the numeric envelope and source-owned silhouette;
- label unowned depth or hidden surfaces as `candidate interpretation`;
- record source, script, Blender, mesh, and review hashes;
- render the unchanged source region beside the Blender candidate using
  Blender only;
- classify the output as `candidate` until Flamestrike decides it;
- preserve an approved component byte-for-byte or by recorded geometry
  signature in every later stage.

Technical checks cannot grant visual approval.
