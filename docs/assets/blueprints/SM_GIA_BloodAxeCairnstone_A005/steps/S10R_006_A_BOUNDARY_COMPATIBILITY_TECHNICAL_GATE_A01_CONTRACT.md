## Exact Compatibility Definitions Proposed For Approval

These definitions apply only if Flamestrike later approves this contract:

### Target-Boundary Identity

- outer N3: `abs(x / 70)^3 + abs(y / 55)^3 = 1`;
- inner K80: `abs(x / 56)^3 + abs(y / 44)^3 = 1`;
- shared mathematical construction origin: `(0,0)`;
- shared declared construction-axis orientation; and
- homothetic scale: `56/70 = 44/55 = 0.8`.

The audit may replay the analytic containment fact that scaling an N3 boundary
point by `0.8` produces a K80 boundary point and that the K80 point evaluates
to `0.8^3 = 0.512` in the N3 equation. This proves strict abstract-boundary
containment only. It does not create a field, center, pivot, source relation,
or geometry.

### Comparable Registered Boundary

Two boundary records are comparable only if both have:

- an approved coordinate value in the same named frame;
- an approved shared semantic boundary or correspondence ID;
- an approved orientation/sector role; and
- a stored exact representation suitable for direct equality testing.

Source-view proximity, visual alignment, nearest-neighbor matching, inferred
cross-view pairing, normalization by panel size, or an assumed transform does
not make records comparable.

### Exact Agreement

For comparable registered target-space boundaries, exact agreement means
identical stored target-coordinate values at the same registered boundary ID.
No visual tolerance, nearest match, averaging, snapping, fitting, or epsilon
chosen after seeing the result is permitted.

If no common exact representation exists, the result is missing authority,
not disagreement. If comparable stored values exist and differ, the result is
an explicit boundary mismatch.

### Field Non-Creation

The normalized radial domain `v in [0,1]` may be recorded as the approved
conditional rule's declared domain. Execution may not instantiate `v`, emit a
field coordinate, rasterize or sample the domain, interpolate a surface,
shade an area, fill a footprint, or create topology or geometry.
