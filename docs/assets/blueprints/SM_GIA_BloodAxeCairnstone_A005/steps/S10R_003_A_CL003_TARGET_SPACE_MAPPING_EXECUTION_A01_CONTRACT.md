## K80 Target-Boundary Lock

The approved target boundary is:

`abs(x / 56)^3 + abs(y / 44)^3 = 1`

Registered axis endpoints:

- positive X: `(56,0)`;
- negative X: `(-56,0)`;
- positive Y: `(0,44)`; and
- negative Y: `(0,-44)`.

Approved sector directions:

- top: negative X to positive X through positive Y;
- right: positive Y to negative Y through positive X;
- bottom: positive X to negative X through negative Y; and
- left: negative Y to positive Y through negative X.

The target origin is a mathematical construction convention only. It is not a
source center, component center, pivot, placement measurement, or claim about
physical C-003 size.

## Pre-Registered Mapping Rule

### Rank Parameter

For each sector independently:

- sample count `n = 4`;
- local rank `r = 1,2,3,4` in the exact source order above; and
- normalized parameter `u = r / (n + 1) = r / 5`.

The only permitted `u` values are `0.2`, `0.4`, `0.6`, and `0.8`.

The rule is endpoint-exclusive. Registered axis endpoints bound each sector
but receive no source sample. This avoids inventing endpoint ownership,
snapping adjacent source sectors together, duplicating target coordinates, or
implying a closed contact loop.

### Sector Parameterization

Use these exact angular rules:

- top: `theta = pi * (1 - u)`;
- right: `theta = pi/2 - pi*u`;
- bottom: `theta = 2*pi - pi*u`; and
- left: `theta = 3*pi/2 - pi*u`.

For each permitted `theta`, calculate:

- `x = 56 * sign(cos(theta)) * abs(cos(theta))^(2/3)`;
- `y = 44 * sign(sin(theta)) * abs(sin(theta))^(2/3)`.

Serialize target coordinates to twelve decimal places using the same formula
and precision convention as the approved A02 curve ledger. The independent
auditor must replay the unrounded formula and compare the serialized result.

### Interpretation Boundary

This is an equal-rank parameter rule, not a source-distance fit and not an
equal-arc-length claim. Pixel gaps, RGB values, apparent curvature, and
cross-view observations do not influence `u` or `theta`.

Every produced target record must carry all of these labels:

- `candidate interpretation`;
- `source pixel unchanged`;
- `no physical cross-view pairing`;
- `no source center or pivot claim`;
- `no physical C-003 dimension claim`;
- `no snap-anchor authority`;
- `no geometry-vertex authority`; and
- `pending Flamestrike decision`.
